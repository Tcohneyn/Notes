在 C++ 和虚幻引擎（Unreal Engine）的开发中，将 `OutOverlappingActors` 设计为**输出参数（Output Parameter）**主要有以下几个原因：

### 1. 引用传递（Pass by Reference）

在函数签名中，可以看到它是这样定义的：`TArray<AActor*>& OutOverlappingActors`。

- **符号 `&`** 表示这是一个**引用**。
- 这意味着函数内部对这个数组进行的任何操作（如 `AddUnique`），实际上都是在直接操作调用者（Caller）传进来的那个原始变量。
- 这使得函数能够将计算结果（搜寻到的 Actor 列表）“带回”给调用方。

### 2. 虚幻引擎的蓝图映射机制

由于这个函数是在 `UAuraAbilitySystemLibrary`（全局函数库）中定义的，它通常会暴露给蓝图使用。

- **非 const 引用 = 蓝图输出引脚**：在虚幻引擎中，如果一个参数是“非 const 引用”（即没有 `const` 修饰符且带 `&`），蓝图系统会自动将其识别为一个**输出引脚（Output Pin）**。
- **直观性**：你在蓝图中调用此节点时，会看到右侧有一个名为 `Out Overlapping Actors` 的引脚，你可以直接从这里连线获取搜寻到的敌人列表。

### 3. 性能优化（避免不必要的拷贝）

虽然现代 C++ 有“返回值优化（RVO）”，但在虚幻引擎的传统架构中，使用引用作为输出参数是一种非常稳健的做法：

- **减少内存开销**：如果函数直接 `return TArray<AActor*>`，在某些情况下可能会发生数组的拷贝。而使用引用传递，函数只是在现有的内存地址上填充数据，效率更高。
- **复用内存**：调用者可以预先创建一个数组，并多次调用此函数来更新它，而不需要每次都销毁并重新创建新的数组。

### 4. 命名约定（Naming Convention）

- **前缀 `Out`**：在虚幻引擎的代码规范中，以 `Out` 开头的参数明确告知其他开发者：“这个变量进函数时可能是空的，出函数时会被填满数据”。这是一种良好的编程习惯，增加了代码的可读性。

### 5. 函数返回值的限制

- 这个函数的返回类型是 `void`（空）。
- 在 C++ 中，一个函数只能有一个返回值。如果你需要函数返回多个结果（比如既要返回是否成功，又要返回 Actor 列表），通常就会使用输出参数。
- 在这个特定的例子中，核心产出就是这个 Actor 数组，因此通过输出参数来提供。

### 总结

`OutOverlappingActors` 就像是一个**“空篮子”**，由你（调用者）提着交给函数。函数在空间里搜寻一番后，把找到的“果实”（存活的 Actor）放进你的篮子里。当你拿回篮子时，里面就装满了你需要的数据。
