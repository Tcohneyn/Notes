## ⚠️ 视频/图形设置实现前的技术挑战

在开始实现“视频”选项卡下的各项图形设置（如窗口模式和分辨率）之前，需要首先解决四个关键的技术挑战，主要涉及数据类型不兼容、设置间的依赖关系和编辑条件的管理。

------

### 1. 🛑 挑战一：窗口模式的 Enum 与 String 不兼容

- **问题描述：**
  - Unreal Engine **内置** 了获取和设置 **窗口模式** (`EWindowMode` enum) 的便利函数（Getter/Setter）。
  - 但现有的设置数据对象 (`List Data Object_String`) 只能处理 **字符串** 类型的数据选项。
  - **枚举 (`Enum`) 类型** (`EWindowMode`) **无法自动转换** 为 **字符串 (`String`)**，导致无法使用现有的数据对象结构来绑定这些内置函数。
- **解决方案需求：**
  - 需要创建一个 **新类型的数据对象**，专门用于添加和处理 **Enum 选项**，并手动处理 Enum 到 String 的转换逻辑。

------

### 2. 🧱 挑战二 & 三：数据依赖 (Data Dependency) 与编辑条件 (Edit Conditions) 框架

挑战二和三将通过构建一个 **通用框架** 来解决，以管理设置的交互性。

- **挑战二：数据依赖（Data Dependency）**
  - **问题描述：** **屏幕分辨率** 的可用选项和可编辑状态**依赖于** **窗口模式** 的当前值。例如，修改窗口模式可能会影响分辨率列表。
- **挑战三：多重编辑条件（Multiple Edit Conditions）**
  - **问题描述：** **屏幕分辨率** 设置是否可编辑（`Editable`）需要满足多个条件：
    1. **条件一（功能依赖）：** 当 **窗口模式** 设置为 **“无边框窗口”** (`Borderless Window`) 时，屏幕分辨率应**不可编辑**（因为此时应使用最大允许分辨率）。
    2. **条件二（避免编辑器问题）：** 为了避免在编辑器内修改值时（尤其是在使用双显示器时）破坏编辑器设置，**窗口模式** 和 **屏幕分辨率** 应**只在打包构建版本中可编辑** (`Package Build Only`)。
- **解决方案需求：**
  - 需要一个 **通用框架**，能够为设置添加 **多个编辑条件** 并定义它们之间的 **数据依赖关系**，确保设置能根据其他设置的值动态调整其可用性。

------

### 4. 🚫 挑战四：禁用时强制设置值 (Disabled For Set Value)

- **问题描述：** 当某个设置因不满足编辑条件而**被禁用**时，它的值需要被 **强制设置** 为一个特定的默认值。
- **实例：** 当 **窗口模式** 被设置为 **“无边框窗口”** 时（导致屏幕分辨率被禁用），分辨率的值必须被 **强制覆盖** 为 **最大允许分辨率**，无论用户之前设置了什么值。
- **解决方案需求：**
  - 该框架不仅需要定义禁用条件，还需要定义一个 **“禁用时强制设置的值”**，以确保逻辑上的正确性。

------

**总结：** 下一阶段的工作将从解决第一个挑战——创建一个新的 **Enum 数据对象** 开始。
