在本节视频课程中，我们首先修复了一个关于加载界面通知机制的逻辑漏洞，随后详细讲解了如何在蓝图中实现**自动滚动的鸣谢名单（Credits Screen）**。

以下是详细的内容大纲总结：

------

### 一、 修复加载子系统逻辑 Bug

在之前的逻辑中，如果我们在编辑器设置中关闭了“显示加载界面”，会导致主 UI 无法弹出的问题。

- **问题根源**：`NotifyLoadingScreenVisibilityChanged` 函数原本被放在 `TryRemoveLoadingScreen` 中。如果系统决定不显示加载界面，则不会创建 Widget，也就永远不会调用移除函数，导致通知永远无法发出。
- **修复方案**：将该通知函数的调用时机移动到 `TryUpdateLoadingScreen` 中。
  - **逻辑调整**：当 `ShouldShowLoadingScreen()` 返回 `false` 时（无论是因为加载结束还是配置关闭），立即触发通知。
- **结果验证**：修复后，即使不显示加载界面，玩家控制器也能正确接收到“加载完成”的信号并推入主菜单 UI。

------

### 二、 鸣谢界面内容生成

为了演示滚动效果，我们在 `BP_CreditScreen` 中动态生成了大量的测试文本。

1. **初始化变量**：创建字符串变量 `CreditsDisplayString`。
2. **重写 `OnActivated` 事件**：当界面被推入视口时触发逻辑。
3. **循环填充内容**：
   - 使用 **For Loop**（从 1 到 100）模拟大量鸣谢名单。
   - 通过 **Append** 节点将“职位标题”、“人员姓名”及循环索引拼接。
   - 使用 `Shift + Enter` 插入换行符，使内容具有纵向深度。
4. **文本更新**：将最终生成的长字符串设置到 **Common Rich Text Block** 中。

------

### 三、 实现自动滚动逻辑

这是本节的核心部分，通过定时器动态改变滚动条的偏移量。

- **重置起始位置**：在滚动开始前，调用 `Set Scroll Offset` 将其重置为 **0**，确保每次打开都从头播放。
- **设置定时器 (Timer)**：
  - 使用 **Set Timer by Event**，设置间隔为 **0.05 秒** 并开启 **Looping**（循环）。
  - 创建自定义事件 `ScrollCredits`。
- **位移逻辑**：
  1. 获取当前滚动框的 `Get Scroll Offset`。
  2. 累加一个 `Scroll Speed`（滚动速度）变量（初始值设为 1）。
  3. 调用 `Set Scroll Offset` 更新位置。

------

### 四、 终止逻辑与 UI 美化

为了确保资源不被浪费且界面整洁，我们添加了检查机制。

- **终点检测**：
  - 获取滚动框的终点位置 `Get Scroll Offset of End`。
  - 使用 **Branch** 分支判断：如果当前偏移量 $\ge$ 终点偏移量，则执行终止。
- **清理定时器**：调用 `Clear and Invalidate Timer by Handle`，停止持续的滚动计算。
- **视觉优化**：
  - 在 Designer（设计器）界面中，将滚动框（Scroll Box）的 **Scroll Bar Visibility** 设置为 **Hidden**（隐藏）。
  - 这样可以获得更像电影结尾的纯净滚动体验。

------
