### 一、 基础布局与文本样式微调

- **文本控件配置：** 将显示名称的文本块（Common Text Block）设置为“填充（Fill）”，并将水平和垂直对齐方式均设为居中。
- **应用样式：** 指定预定义的文本样式 `list_entry_default`，并设置预览文本为 "Setting name"。
- **位置平衡：**
  - 为了防止按钮过于靠右，在水平框（Horizontal Box）末尾添加了一个 **Spacer（间览器）**，宽度设为 50。
  - 为文本块添加了 220 的右侧内边距（Padding），以确保操作名称和交互按钮之间有美观的间距。
- **初步预览：** 在数据资产中关联该蓝图并运行项目，确认条目在“选项-控制”界面中排布正确。

### 二、 修复 C++ 变量绑定

- **图像控件重命名：** 发现按钮内的 `CommonLazyImage` 并没有绑定到 C++ 变量。
- **操作步骤：** 回到 C++ 代码中复制变量名 `CommonLazyImage_ButtonImage`，然后在蓝图层级中将其重命名，确保 C++ 逻辑可以在运行时控制该图像。

### 三、 实现编辑器预览图像系统

由于运行时按键图标会由代码动态生成，为了在编辑器中直观看到效果，需要建立一套**预览系统**：

- **自定义变量：** 创建了一个公开的变量 `Preview_ButtonImage`（类型为 Lazy Texture）。
- **逻辑控制：** 在 `PreConstruct`（预构造）事件中使用 **`IsDesignTime`** 分支。
  - 仅当在编辑器设计环境下且预览变量有效时，才调用 `SetBrushFromLazyTexture` 来设置图标。
- **目的：** 这种做法可以避免在编辑器中手动设置的图片干扰运行时的动态逻辑。

### 四、 按钮图标尺寸自适应

- **尺寸重写：** 默认图标尺寸可能不一。为了统一视觉效果，引入了 `ButtonImageSize`（Float 类型）变量。
- **功能实现：** 在预构造事件中调用 `SetDesiredSizeOverride` 函数。
- **优化：** 将默认尺寸设为 35，但在按键条目蓝图中将其增加到 **60**，以获得更好的视觉清晰度。

### 五、 完成重置按钮配置

- **静态图标分配：** 针对“重置按键”按钮，由于其功能固定，直接通过 `Preview_ButtonImage` 变量指定了 `UI_Clear_Icon`。
- **对齐与大小：** 同样设置水平/垂直对齐为居中，并将图像尺寸设为 60。

- **当前成果：** 列表条目的 UI 布局已经美观且符合规范，具备了重映射按钮和重置按钮。
