# 敌人默认属性系统构建（Gameplay Effects）

—— 标题大纲式技术总结

## **1. 项目结构整理：区分 Aura 与 Enemy 的默认属性**

### 1.1 创建独立目录结构

- 现有 Aura 的默认属性 Gameplay Effects
- 新增 Enemy 文件夹，专用于敌人属性

### 1.2 移动文件

- 所有 Aura 专用 GE 移入 Aura 文件夹
- 为 Enemy 创建新的属性 GE 资源空间

------

## **2. 为每个敌人职业创建 Primary Attributes Gameplay Effect**

### 2.1 创建 3 个独立 GE（按职业划分）

- GE_PrimaryAttributes_Elementalist
- GE_PrimaryAttributes_Warrior
- GE_PrimaryAttributes_Ranger

### 2.2 用途

为每个职业提供不同的 **初始主属性配置**（力量、智力、敏捷等基础属性）

------

## **3. Secondary Attributes：共享 or 分职业？（系统设计决策点）**

### 3.1 二次属性关系可共享

- 默认 Aura 的 Secondary GE 已包含属性关系：
  - armor ↔ resilience
  - 其他 derived attributes 的 coefficient、pre/post multiplier、additive 设定

### 3.2 若需要职业差异化，可创建 3 份 GE

- 但代价高，需要重新配置属性关系
- RPG 项目中常见需求：职业间不同成长曲线、曲线表等

### 3.3 教程选择：统一共享

- 将 Aura 的 “Aura Secondary Attributes” 改名为
   ✔ **GE_SecondaryAttributes**
- 所有角色 / 敌人通用

------

## **4. Vital Attributes：同样可共享**

### 4.1 Vital（生命、法力、耐力类）通常无需职业差异

- 和 Secondary 类似，可统一共享

### 4.2 教程选择：统一共享

- 将 “Aura Vital Attributes” 改名为
   ✔ **GE_VitalAttributes**
- 所有角色 / 敌人通用

### 4.3 移动文件

- GE_SecondaryAttributes、GE_VitalAttributes → Default Attributes 根目录
- 方便所有系统统一使用

------

## **5. 在 Character Class Info 中绑定 Gameplay Effects**

### 5.1 配置每个职业的 Primary GE

- Elementalist → GE_PrimaryAttributes_Elementalist
- Warrior → GE_PrimaryAttributes_Warrior
- Ranger → GE_PrimaryAttributes_Ranger

### 5.2 配置共享 Secondary / Vital GE

- Secondary → GE_SecondaryAttributes
- Vital → GE_VitalAttributes

### 5.3 作用

让系统能按“角色职业”自动初始化敌人属性数据。

------
