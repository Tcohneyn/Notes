这段视频演示了如何使用之前创建的技能球按钮（Spell Globe Button）构建**攻击系技能树（Offensive Spell Tree）**控件。

以下是详细的内容大纲总结：

### 一、 控件初始化与基础布局

- **创建控件**：在 `SpellMenu` 文件夹下创建新的用户控件 `WBP_OffensiveSpellTree`，继承自自定义的 `AuraUserWidget`。
- **尺寸控制（Size Box）**：
  - 使用 **Size Box** 作为根节点，以便精确控制整个技能树的宽高。
  - **动态调整**：在 `PreConstruct` 事件中通过变量（Box Width/Height）调用 `Set Width Override`，方便后续根据 UI 需求调整整体尺寸（初始设定约为 525x350）。
  - **预览优化**：将设计视图从 `Fill Screen` 改为 `Desired`，以实时预览控件的真实大小。

### 二、 使用 Wrap Box 构建三列结构

- **Wrap Box 根节点**：
  - 将 `Wrap Box Root` 的水平和垂直对齐方式设为 `Center`。
  - **垂直排列逻辑**：将 `Orientation` 属性从默认的 Horizontal 改为 **Vertical**。这样子组件会在垂直方向排列，超出高度后再向右侧换列。
- **列容器设计**：
  - 虽然可以直接在根 Wrap Box 堆叠，但为了方便管理，作者在内部创建了三个独立的 `Wrap Box`（命名为 `Col_1/2/3`）。
  - 每一列的 Wrap Box 同样设置为 **Vertical** 布局，代表一类技能的升级路径。

### 三、 技能球与连接线的组织

- **添加技能球**：将 `SpellGlobe_Button` 拖入各列中。
- **添加连接线（Images）**：
  - 在每两个技能球之间插入一个名为 `Line` 的图像组件（Image）。
  - **视觉调整**：将连接线高度设为较小值（如 15），营造出技能点之间被线条连接的视觉效果。
- **重复结构**：通过复制粘贴，每列形成“球-线-球-线-球”的结构，每列包含 3 个技能球。

### 四、 间距调整与细节优化

- **使用 Spacer（间距器）**：
  - 为了在三列之间制造空隙，在列容器之间插入 **Spacer** 组件。
  - **高度触发逻辑**：由于根容器是垂直 Wrap Box，必须给 Spacer 设置足够的高度（如 100），确保它能占据足够的垂直空间从而将下一列推向右侧。
  - **宽度设定**：将 Spacer 的 X 值设为 50，实现清晰的列间距。

### 五、 实时预览与验证

- **HUD 整合测试**：将制作好的 `Offensive Spell Tree` 临时拖入 HUD 的 Overlay 控件中。
- **运行测试**：在编辑器中点击 Play，在实际游戏画面中观察技能树的大小是否合适。
